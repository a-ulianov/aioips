# Установка

На этой странице — как поставить `aioips` в чистое окружение и убедиться, что всё работает.

## Быстрая установка из PyPI

```bash
pip install aioips
```

Этого достаточно для использования клиента. Ниже — установка из исходников (для разработки и
запуска тестов).

## Требования

| Что | Версия / условие |
|---|---|
| Python | **3.13 или новее** |
| ОС | Windows, Linux, macOS (клиент кроссплатформенный) |
| Доступ | сетевой доступ к серверу IPS Web API |

!!! note "Почему именно 3.13"
    `aioips` использует современный синтаксис типов (`list[X]`, `X | None`) и строгую проверку
    `mypy`. На более ранних версиях Python установка не поддерживается.

Проверьте версию интерпретатора:

```bash
python --version
# Python 3.13.x
```

## Шаг 1. Получите исходники

```bash
git clone https://github.com/a-ulianov/aioips.git
cd aioips
```

## Шаг 2. Создайте виртуальное окружение

Изолированное окружение держит зависимости проекта отдельно от системного Python.

=== "Windows (PowerShell)"

    ```powershell
    python -m venv .venv
    .venv\Scripts\Activate.ps1
    ```

=== "Linux / macOS"

    ```bash
    python -m venv .venv
    . .venv/bin/activate
    ```

!!! tip "Каталог окружения"
    Используйте именно `.venv/` в корне проекта — это соглашение репозитория, и каталог уже
    добавлен в `.gitignore`.

## Шаг 3. Установите пакет

Установка в режиме «editable» (`-e`) — изменения в исходниках сразу подхватываются без переустановки.

```bash
pip install -e ".[dev]"
```

Что устанавливается:

- **основные зависимости** — `aiohttp` (async HTTP-транспорт), `pydantic` v2 и
  `pydantic-settings` (схемы и конфигурация), `tenacity` (повторы транзиентных ошибок);
- **экстра `dev`** — инструменты разработки: `ruff`, `mypy`, `pytest`.

Если инструменты разработки не нужны (вы просто пользуетесь клиентом), достаточно базовой установки:

```bash
pip install -e .
```

!!! note "Менеджер пакетов"
    Вместо `pip` можно использовать `uv` (он быстрее): `uv pip install -e ".[dev]"`. Источник
    зависимостей один — `pyproject.toml`; файлов `requirements.txt`/`setup.py` в проекте нет.

## Шаг 4. Проверьте установку

Импорт публичного API должен пройти без ошибок:

```bash
python -c "from aioips import IPSClient, IPSConfig; print('ok')"
# ok
```
