# Быстрый старт

Эта страница — самый короткий путь от установленного пакета до реального ответа сервера IPS.
Мы создадим конфигурацию, откроем клиент, узнаем, кто мы, и заглянем в метамодель.

## Предусловия

- Пакет [установлен](installation.md).
- [Конфигурация](configuration.md) готова: задан `base_url` и авторизационные данные (логин/пароль
  или токен). Удобнее всего держать их в `.env`.

## Полный пример

```python
import asyncio

from aioips import IPSClient, IPSConfig


async def main() -> None:
    # 1. Конфигурация подключения и авторизации.
    #    Можно задать явно, как здесь, или прочитать из окружения/.env: IPSConfig()
    config = IPSConfig(
        base_url="http://your-ips-host:8080",
        login_name="your-login",
        password="...",          # лучше — через переменную окружения IPS_PASSWORD
        role_name="Администратор",
    )

    # 2. Открываем клиент через async with — он сам закроет сессию на выходе.
    async with IPSClient(config=config) as ips:
        # 3. «Кто я»: проверяем, что сессия действительна.
        #    Здесь же клиент лениво получает JWT-токен (первый авторизованный запрос).
        me = await ips.user_info()
        print("Вход выполнен как:", me.login_name)
        print("Администратор:", me.is_admin)

        # 4. Заглядываем в метамодель: все типы объектов IPS.
        types = await ips.object_types()
        print("Типов объектов в базе:", len(types))


if __name__ == "__main__":
    asyncio.run(main())
```

Ожидаемый вывод (значения зависят от вашей базы):

```text
Вход выполнен как: your-login
Администратор: True
Типов объектов в базе: 137
```

## Разбор по шагам

### 1. Конфигурация

`IPSConfig` собирает всё, что нужно для подключения. Если параметры лежат в `.env` или в
переменных окружения, конфигурацию можно создать без аргументов:

```python
config = IPSConfig()
```

Подробности — на странице [Конфигурация](configuration.md).

### 2. Клиент как контекстный менеджер

`IPSClient` всегда используется через `async with`: на входе он поднимает HTTP-сессию, на выходе
аккуратно её закрывает. Поэтому весь код, делающий запросы, живёт **внутри** блока `async with`.

```python
async with IPSClient(config=config) as ips:
    ...  # здесь доступны все методы: ips.user_info(), ips.object_types(), ...
```

### 3. Первый запрос и авторизация

Токен не запрашивается заранее — он берётся **лениво**, при первом методе, которому нужна
авторизация (здесь это `user_info()`). Дальше клиент обновляет токен сам. Вам не нужно вызывать
вход вручную. Как именно это работает — на странице [Авторизация](authentication.md).

`user_info()` возвращает объект `CurrentUserInfo` с полями `login_name`, `is_admin`, `session_id`
и другими — удобно для проверки, что вход действительно состоялся.

### 4. Запрос к метамодели

`object_types()` возвращает список всех типов объектов, определённых в базе IPS. Это «чтение»
без побочных эффектов — безопасный способ убедиться, что клиент видит данные сервера.

## Частые вопросы

!!! question "Нужно ли логиниться вручную?"
    Нет. Клиент получает и обновляет JWT-токен автоматически при первом авторизованном запросе.

!!! question "Что, если параметры в `.env`?"
    Тогда `config = IPSConfig()` — без аргументов. Адрес и креды подхватятся автоматически.

!!! warning "Не вызывайте методы вне `async with`"
    После выхода из блока `async with` сессия закрыта, и запросы выполнить нельзя.
