# Файлы (files, file_systems)

**Что за раздел.** Работа с **файловыми атрибутами** объектов: чтение и выгрузка содержимого файлов,
загрузка новых файлов через временное хранилище и привязка их к объекту. Файлы в IPS — это значения
атрибутов типа «файл» (`ftFile`), хранящиеся во внешнем хранилище (vault), а не в основной БД.

**Когда нужен.** Когда надо скачать файл, прикреплённый к объекту, или загрузить новый. Загрузка —
двухшаговая: сначала `upload_temp_file` помещает файл во временное хранилище и возвращает его имя,
затем `attach_temp_files` привязывает временные файлы к файловым атрибутам объекта. Раздел
[`file_systems`](#file_systems) даёт доступ к локальным дискам сервера.

## files

### Чтение и выгрузка

| Метод | Назначение |
|---|---|
| `file_attributes(object_id)` | Возвращает объект вместе со всеми его файловыми атрибутами и файлами. |
| `object_file_by_name(object_id, file_name)` | Загружает содержимое файла объекта, выбранного по его имени. |
| `object_file_by_blob_id(object_id, blob_id)` | Загружает содержимое файла объекта по идентификатору BLOB-записи. |
| `file_prototypes(object_id)` | Возвращает список файлов-прототипов (шаблонов), доступных для объекта. |

### Идентификаторы и имена файлов

| Метод | Назначение |
|---|---|
| `file_id_by_name(*, file_name=None)` | Возвращает идентификатор файла по его имени из сервиса имён файлов IPS. |
| `object_ids_by_file_name(*, file_name=None)` | Возвращает идентификаторы объектов, владеющих файлом с заданным именем. |
| `file_unique_name(*, file_name=None, id=None)` | Возвращает уникальное имя файла на основе предложенного имени. |
| `next_file_id()` | Возвращает следующий свободный идентификатор файла из id-пространства IPS. |

### Загрузка и привязка (временное хранилище)

| Метод | Назначение |
|---|---|
| `upload_temp_file(file_data, file_name, *, is_already_packed=None)` | Загружает файл во временное хранилище сервера и возвращает его имя (МУТИРУЮЩАЯ). |
| `attach_temp_files(object_id, attachments)` | Прикрепляет загруженные временные файлы к файловым атрибутам объекта (МУТИРУЮЩАЯ). |
| `delete_temp_file(temp_file_name)` | Удаляет ранее загруженный временный файл из временного хранилища (МУТИРУЮЩАЯ). |

### Прямые мутации файлов объекта (обратимый цикл)

Прямое прикрепление/замена/удаление файла в файловом атрибуте (`ftFile`) объекта — без временного
хранилища. **Важно (проверено на проде):** объект должен быть на checkout, а `object_id` — это id
**рабочей копии** (результат `object_check_out`, на проде отрицательный), НЕ базовый `ObjectID`.

| Метод | Назначение |
|---|---|
| `add_object_file(object_id, attribute_id, file_data, file_name, file_type, modify_date_time, *, real_file_size=None)` | Прикрепляет НОВЫЙ файл к файловому атрибуту (multipart → `BlobFileAttributeInfo`). |
| `update_object_file(object_id, attribute_id, blob_id, file_data, file_name, modify_date_time, *, real_file_size=None)` | Заменяет содержимое уже прикреплённого файла (multipart). |
| `delete_object_file(object_id, *, attribute_id, blob_id, confirm=False)` | Удаляет файл из атрибута. `confirm=True` (РАЗРУШАЮЩАЯ; обратная к add). |
| `update_object_file_info(object_id, body)` | Правит метаданные файла (имя/комментарий) без замены содержимого. |

### Табличные выборки метаданных файлов (чтение через POST)

POST-эндпоинты, но это **чтения**: возвращают `DataTableDto` (`columns` + `rows`) или дерево файлов.
Не изменяют данные.

| Метод | Назначение |
|---|---|
| `get_file_names_table(file_names)` | Таблица метаданных файлов по списку имён (`DataTableDto`). |
| `get_files_table(file_ids)` | Таблица метаданных файлов по списку id файлов (`DataTableDto`). |
| `get_files_table_by_fields(body)` | Таблица файлов объектов с выбором колонок (`ObjectIdsWithColumnsDto`). |
| `get_files_table_all_attributes(body)` | Таблица файлов объектов со всеми атрибутами (`ObjectIdsWithColumnsFileNameDto`). |
| `get_files_table_with_snapshot_ids(body)` | Таблица файлов объектов в контексте снимков (`ObjectSnapshotIds`). |
| `get_file_name_table()` | Таблица имён файлов (без параметров; зависит от состояния сессии сервера). |
| `check_unique_file_names(file_id, file_names)` | Проверяет уникальность имён файлов, возвращает рекомендации. |
| `object_files_with_composition(object_id, body=None)` | Дерево файлов объекта с разворачиванием состава (`ObjectFilesTreeNodeDto`). |

## file_systems

**Что за раздел.** Информация о файловой системе сервера IPS.

| Метод | Назначение |
|---|---|
| `find_directories(params)` | Метод поиска каталогов в файловой системе сервера IPS |
| `find_files(params)` | Метод поиска файлов в файловой системе сервера IPS |
| `is_directory_exists(path)` | Метод проверки существования каталога в файловой системе сервера IPS |
| `local_drives()` | Метод получения списка локальных дисков сервера IPS |
