# Атрибуты и типы данных

Атрибуты — это характеристики объекта: обозначение, наименование, масса, дата, ссылка на другой
объект, прикреплённый файл. Всё содержательное «наполнение» объекта хранится в его атрибутах.

## Простыми словами

Если объект — это карточка детали, то атрибуты — это поля в карточке:

- «Обозначение» = `550.07.305` (строка),
- «Масса» = `12.4` (число),
- «Дата создания» = `2026-06-24` (дата),
- «Архив» = ссылка на объект-архив (ссылка на объект),
- «Чертёж» = прикреплённый PDF-файл (файл).

У каждого атрибута есть **тип данных** (что именно в нём лежит) и **режим множественности**
(одно значение или несколько). Зная тип, вы знаете, как правильно читать значение.

## Точные детали

### Типы данных (`FieldTypes`)

Тип атрибута приходит в его метаданных и определяет, как хранится и читается значение. В `aioips`
это перечисление `FieldType` (модуль `aioips.common.enumerations`). Полный набор — 18 значений:

| `FieldType` | Значение в API | Что это |
|---|---|---|
| `STRING` | `ftString` | Строка до 450 символов |
| `INTEGER` | `ftInteger` | Целое (Int64) |
| `DOUBLE` | `ftDouble` | Число с плавающей точкой |
| `DATE_TIME` | `ftDateTime` | Дата/время (**хранится в UTC**) |
| `BOOLEAN` | `ftBoolean` | Логическое значение |
| `MEMO` | `ftMemo` | Длинный текст |
| `MEASURED` | `ftMeasured` | Измеряемая величина (число + единица измерения) |
| `OBJECT_LINK` | `ftObjectLink` | Ссылка на объект; значение = **id объекта-цели** |
| `OBJECT_LINK_BY_ID` | `ftObjectLinkByID` | Ссылка на объект по id версии |
| `EXTERNAL_LINK` | `ftExternalLink` | Внешняя ссылка (URL/путь) |
| `FILE` | `ftFile` | Файл во внешнем хранилище/vault (диск X:) |
| `BLOB` | `ftBlob` | Двоичные данные в БД |
| `SHORT_BLOB` | `ftShortBlob` | Небольшой двоичный блок (≤~128 КБ в БД) |
| `GUID` | `ftGuid` | Глобальный идентификатор |
| `PASSWORD` | `ftPassword` | Хэш пароля |
| `AUTO_INC` | `ftAutoInc` | Автоинкрементное значение |
| `SYSTEM` | `ftSystem` | Системный обязательный атрибут |
| `UNKNOWN` | `ftUnknown` | Тип не определён |

!!! warning "Ссылку на объект читайте как id, а не как объект"
    Значение атрибута-ссылки (`ftObjectLink` / `ftObjectLinkByID`) — это **id объекта-цели**.
    Берите именно id (или GUID) — это дёшево. Не «разворачивайте» ссылку в полный объект без
    необходимости: это грузит весь связанный объект. Так, в частности, устроена привязка
    «документ → Архив»: «Архив» — это атрибут-ссылка (в одной из боевых БД его id = 1029), а не
    состав. Поэтому документы архива и ищут по этой ссылке (см. [Поиск](search.md)).

!!! warning "Даты — в UTC"
    Значения `ftDateTime` хранятся в UTC. Учитывайте это при сравнении и выводе: «локальное»
    представление может отличаться от хранимого. Не сравнивайте «наивные» локальные даты с тем,
    что пришло из IPS, без приведения к одному поясу.

### Множественность (`MultiValueModes`)

Атрибут может содержать одно значение или несколько, и набор значений может быть свободным или
ограниченным списком. В `aioips` — перечисление `MultiValueMode`:

| `MultiValueMode` | Значение в API | Смысл |
|---|---|---|
| `SINGLE_VALUE` | `singleValue` | Ровно одно значение |
| `MULTI_VALUES` | `multiValues` | Несколько произвольных значений |
| `SINGLE_VALUE_FROM_LIST` | `singleValueFromList` | Одно значение из списка допустимых |
| `MULTI_VALUES_FROM_LIST` | `multiValuesFromList` | Несколько значений из списка допустимых |

У режимов `…FromList` значения ограничены набором допустимых (в ядре IPS его отдаёт
`GetPossibleValues()`). Поэтому в `aioips` значения атрибута всегда лежат в **списке** `values`
(даже если значение одно) — это унифицирует одно- и многозначные атрибуты.

### Вычисляемость (`ComputeValueModes`)

Не каждое значение хранится «как есть» — часть вычисляется. Перечисление `ComputeValueMode`:
`NOT_COMPUTABLE` (`notComputableValue`), `STORED` (`storedValue`), `JIT` (`jitValue` — считается
по запросу), `INDEX` (`indexValue`). Это влияет на возможность записи: вычисляемое значение
обычно нельзя задать напрямую.

### Значения vs описания

Различайте две вещи у атрибута:

- **Значение** — то, что хранится (число, строка, id ссылки).
- **Описание** (description) — человекочитаемое представление значения (например, для ссылки —
  заголовок связанного объекта, для значения из списка — его подпись).

!!! warning "Пустые коллекции приходят как null"
    IPS для пустых списков отдаёт `null`, а не `[]`. Например, `descriptions` может прийти `null`.
    Схемы `aioips` это нормализуют (пустой список вместо `None`), но помните об этом, если читаете
    «сырой» ответ.

## Как это выглядит в коде aioips

Чтение всех атрибутов объекта (это **только чтение**, режим редактирования не нужен):

!!! example "Прочитать атрибуты объекта"
    ```python
    async with IPSClient(config=config) as ips:
        attributes = await ips.object_attributes(102550)  # 102550 = objectID
        # удобная карта «имя → отображаемое значение»
        values = {a.name: a.as_string for a in attributes}
        print(values.get("Обозначение"))
    ```

У каждого `Attribute`: `attribute_id` (тип атрибута), `name`, `as_string` (готовое строковое
представление) и `values` (типизированные значения — их интерпретация зависит от `FieldType`).

Запись значений атрибута — это отдельная история: её можно делать только в правильном режиме
редактирования. См. [Жизненный цикл](lifecycle.md) и руководство
[Редактирование объектов](../guides/editing-objects.md).
