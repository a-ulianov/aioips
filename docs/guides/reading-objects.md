# Чтение объектов

Это руководство — про **безопасные операции только на чтение**: получить объект целиком, узнать
его краткую сводку, прочитать все атрибуты или одно конкретное значение. Ни один из этих методов не
требует извлечения объекта на редактирование (checkout) и ничего не меняет на сервере.

!!! tip "Главное про идентификаторы"
    Все методы ниже принимают **id ОБЪЕКТА** (`objectID` / `F_OBJECT_ID`) — общий для всех версий
    объекта. Если передать id версии (`id` / `F_ID`), сервер вернёт пусто, и метод отдаст `None`.
    Разница между объектом и версией — в концепции [Объект и версия](../concepts/data-model.md).

## Какой метод выбрать

| Метод | Что возвращает | Когда применять |
|---|---|---|
| `object_get(object_id)` | `ObjectDto \| None` — полное описание объекта | Нужен объект целиком |
| `object_info(object_id)` | `QuickObjectInfo \| None` — заголовок, тип, GUID | Нужна лёгкая сводка / проверка существования |
| `object_attributes(object_id)` | `list[Attribute]` — все атрибуты со значениями | Нужны все характеристики разом |
| `object_attribute_values(object_id, attribute_id)` | `list \| None` — «сырые» значения одного атрибута | Нужны только значения одной характеристики |
| `object_attribute_as_string(object_id, attribute_id)` | `str \| None` — готовая строка | Нужно отображаемое значение строкой |

---

## Задача 1. Получить объект целиком

**Когда нужно:** известен числовой id объекта, и нужно его полное описание (заголовок, тип, версия).

### Решение

1. Откройте клиент.
2. Вызовите `object_get(object_id)`, передав **id объекта**.
3. Проверьте результат на `None` (объект может не найтись).

### Полный пример

```python
import asyncio

from aioips import IPSClient, IPSConfig


async def main() -> None:
    config = IPSConfig()  # параметры из окружения / .env
    async with IPSClient(config=config) as ips:
        obj = await ips.object_get(102550)  # 102550 — это objectID, не версия
        if obj is None:
            print("Объект не найден")
            return
        print("Заголовок:", obj.caption)
        print("Тип объекта:", obj.object_type)
        print("id объекта:", obj.object_id, "| id версии:", obj.id)


if __name__ == "__main__":
    asyncio.run(main())
```

!!! warning "object_get возвращает None, а не исключение"
    По умолчанию при отсутствии объекта метод вернёт `None`, а не поднимет ошибку. Всегда проверяйте
    результат. Если нужно именно исключение (404), передайте `throw_not_found=True`:
    `await ips.object_get(102550, throw_not_found=True)`.

!!! warning "Передали id версии — получили None"
    Если `object_get` упорно возвращает `None`, хотя объект точно есть, — почти наверняка вы передали
    **id версии** (`F_ID`) вместо **id объекта** (`F_OBJECT_ID`). Это самая частая ошибка.

## Задача 2. Узнать краткую сводку (или просто проверить, что объект есть)

**Когда нужно:** не нужно тащить весь `ObjectDto` — достаточно заголовка и типа, или надо лишь
проверить существование объекта. `object_info` дешевле и быстрее, чем `object_get`.

### Решение

1. Вызовите `object_info(object_id)`.
2. Результат `None` означает, что объекта нет.

### Полный пример

```python
async with IPSClient(config=config) as ips:
    info = await ips.object_info(102550)
    exists = info is not None
    if info is not None:
        print(info.caption, info.object_type_id, info.version_guid)
```

## Задача 3. Прочитать все атрибуты объекта

**Когда нужно:** нужны все характеристики объекта сразу (обозначение, наименование, масса, ссылки и
т.п.) — например, чтобы построить словарь «имя → значение».

### Решение

1. Вызовите `object_attributes(object_id)`.
2. Пройдитесь по списку `Attribute`. У каждого элемента есть `attribute_id` (id типа атрибута),
   `name`, `as_string` (готовая строка) и `values` (список значений).

### Полный пример

```python
async with IPSClient(config=config) as ips:
    attributes = await ips.object_attributes(102550)

    # Словарь "имя атрибута -> строковое значение"
    by_name = {a.name: a.as_string for a in attributes}
    print("Обозначение:", by_name.get("Обозначение"))

    # Или доступ по id типа атрибута
    by_id = {a.attribute_id: a.values for a in attributes}
    print("Значения атрибута 9:", by_id.get(9))
```

!!! tip "values vs as_string"
    `as_string` — это то, как IPS форматирует значение для показа (с учётом типа, единиц, нескольких
    значений). `values` — список «сырых» значений. Для атрибута-ссылки (`ftObjectLink`) элемент
    `values` — это **id связанного объекта**, а не его название.

## Задача 4. Прочитать значения одного атрибута

**Когда нужно:** интересует ровно одна характеристика, и нужны её значения (атрибут может быть
множественным, поэтому всегда список).

### Решение

1. Узнайте id типа нужного атрибута (например, через метаданные или из списка `object_attributes`).
2. Вызовите `object_attribute_values(object_id, attribute_id)`.

### Полный пример

```python
async with IPSClient(config=config) as ips:
    # attribute_id=9 — например, "Обозначение"
    values = await ips.object_attribute_values(102550, 9)
    if values is None:
        print("Атрибут отсутствует у объекта")
    else:
        print("Значения:", values)  # список; для ftObjectLink — id связанных объектов
```

!!! warning "Множественный атрибут — всегда список"
    Даже если у атрибута одно значение, метод вернёт список (например `["550.07.305"]`). Не
    забывайте брать `values[0]`, когда уверены в единственности.

---

## Задача 5. Получить отображаемое значение строкой

**Когда нужно:** нужна именно готовая строка для показа пользователю — IPS сам отформатирует значение
(с единицами измерения, объединением нескольких значений и т.д.).

### Решение

1. Вызовите `object_attribute_as_string(object_id, attribute_id)`.

### Полный пример

```python
async with IPSClient(config=config) as ips:
    text = await ips.object_attribute_as_string(102550, 9)
    print(text)  # например: "550.07.305"
```

!!! warning "Это POST, но всё равно только чтение"
    Несмотря на то что под капотом метод использует HTTP POST (так устроен этот эндпоинт IPS),
    он **ничего не меняет** и не требует checkout. Возвращает `None`, если значения нет.
