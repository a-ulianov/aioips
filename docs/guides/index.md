# Руководства

Этот раздел — практический. Если [Концепции](../concepts/index.md) объясняют, **как устроена IPS**,
то руководства показывают, **как решать конкретные задачи** с помощью `aioips`: получить объект,
изменить его атрибуты, найти нужные данные, скачать файл, связать объекты, корректно обработать ошибку.

Каждое руководство построено одинаково:

1. **Задача** — что мы хотим сделать и когда это нужно.
2. **Решение** — пошаговый, пронумерованный план.
3. **Полный пример** — рабочий код, который можно скопировать и запустить.
4. **Подводные камни** — блоки `!!! warning` с граблями, на которых легко споткнуться.

!!! tip "Прежде чем начинать"
    Все примеры предполагают, что у вас готова [конфигурация](../getting-started/configuration.md)
    и вы умеете открывать клиент (см. [Быстрый старт](../getting-started/quickstart.md)). Код в
    примерах живёт **внутри** блока `async with IPSClient(config=config) as ips:`.

## Карта руководств

| Руководство | Задача | Когда читать |
|---|---|---|
| [Чтение объектов](reading-objects.md) | Получить объект по id, прочитать его атрибуты, значения и краткую сводку | Когда нужно **прочитать** данные одного известного объекта |
| [Редактирование объектов](editing-objects.md) | Полный цикл записи: создать → зафиксировать → извлечь на правку → записать → закоммитить; правка существующего; удаление | Когда нужно **изменить** данные (самое важное руководство по записи) |
| [Поиск и выборки](searching.md) | Найти объекты по условиям на атрибуты, перечислить версии, развернуть состав | Когда id заранее **неизвестны** и объекты надо найти |
| [Файлы](files.md) | Скачать файл объекта, загрузить временный файл, прикрепить его к атрибуту | Когда работаете с вложениями (`ftFile`) |
| [Связи и состав](relations.md) | Прочитать, создать и удалить связь «родитель → потомок», читать атрибуты связей | Когда работаете со структурой изделия |
| [Обработка ошибок и повторы](error-handling.md) | Разобраться в иерархии исключений, понять коды (400/401/403/404/409/429/500), узнать про авто-повторы | Всегда — это нужно для надёжного кода |

## С чего начать

- Никогда не работали с `aioips`? Начните с [Чтения объектов](reading-objects.md) — это безопасные
  операции только на чтение.
- Нужно что-то изменить? Сразу читайте [Редактирование объектов](editing-objects.md) **целиком** —
  без понимания цикла `check_out → set → check_in` запись не работает.
- Пишете боевой код? Держите рядом [Обработку ошибок](error-handling.md).

!!! warning "Идентичность объекта и версии — основа всего"
    Большинство ошибок новичка — из-за путаницы между **id объекта** (`objectID` / `F_OBJECT_ID`,
    общий для всех версий) и **id версии** (`id` / `F_ID`). Если что-то возвращает `None` или 404 —
    первым делом проверьте, какой id вы передали. Подробно: [Объект и версия](../concepts/data-model.md).
