# IPS Server Web API — обзор

- Дата: 2026-06-24

Источник истины — OpenAPI-спека сервера (`/swagger/1.0/swagger.json`, Swagger UI на
`/swagger/index.html`).

## Факты

- REST (ASP.NET Core), OpenAPI **3.0.1**, тела — JSON.
- **744 операции / 697 путей** (GET 415, POST 303, DELETE 17, PUT 9).
- Авторизация — **JWT Bearer** (см. [[auth]]).
- Почти всё под префиксом `/core/api/...` (681 путь); ещё 16 — под `/api/...`.
- Это движок самого веб-клиента IPS; входит в базовую поставку (не путать с отдельными
  модулями WebPortal/XmlExchange).

## Крупнейшие разделы (по числу операций)

| Раздел | ~Операций | Назначение |
|---|---:|---|
| `metadata` | 224 | Типы объектов/связей/атрибутов, применяемость — фундамент |
| `objects` | 64 | Объекты: чтение/создание/изменение, атрибуты, версии |
| `security` | 59 | Пользователи, группы, роли, права доступа |
| `imbase` | 44 | Справочная система Imbase |
| `Bridge` | 35 | Сессии, история изменений |
| `files` | 25 | Файлы объектов |
| `relations` | ~28 | Связи между объектами |
| `improjects` | 22 | Проекты Improject |

## Порядок наращивания обёртки

1. `metadata` (типы объектов/атрибутов) — нужны ID/GUID для всего остального. Начато:
   `object_types`.
2. `objects` + `relations` — основная работа с данными.
3. `security` — пользователи/роли/права.
4. Файлы, выборки, документооборот.

## Особенности
Сводка граблей — [[gotchas]]. Ключевое: реальные ответы могут расходиться со схемой (поля
опциональны по факту), кириллица в описаниях спеки повреждена генератором.
